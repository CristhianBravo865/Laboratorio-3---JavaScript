Resumen JavaScript Numbers

Tipo Único de Número

JavaScript usa un solo tipo de número (con o sin decimales).

Ejemplos:

  • let x = 3.14;
  • let y = 3;

Notación Científica

Se pueden escribir números grandes o pequeños usando "e":

  • let x = 123e5; // 12300000
  • let y = 123e-5; // 0.00123

Siempre en 64 bits

Todos los números siguen el formato IEEE 754 de 64 bits.

Precisión de Enteros

Los enteros son exactos hasta 15 dígitos:

  • 999999999999999 -> correcto
  • 9999999999999999 -> error, se redondea

Precisión de Decimales

Las operaciones con decimales no siempre son exactas:

  • 0.2 + 0.1 // no da exactamente 0.3
  • Solución: (0.2 * 10 + 0.1 * 10) / 10;

Suma de Números y Strings

El operador + puede sumar o concatenar:

  • 10 + 20 -> 30 (suma)
  • "10" + "20" -> "1020" (texto)
  • 10 + "20" -> "1020" (texto)

Importante: se evalúa de izquierda a derecha.

Cadenas Numéricas

JavaScript intenta convertir textos numéricos cuando puede:

  • "100" / "10" -> 10
  • "100" - "10" -> 90
  • "100" + "10" -> "10010" (concatenación)

NaN (Not a Number)

NaN aparece si el cálculo no es válido:

  • 100 / "Apple" -> NaN
  • isNaN(x) // para verificar si es NaN
  • typeof NaN -> "number"

Infinity

Infinity aparece si el número es demasiado grande o se divide entre 0:

  • 2 / 0 -> Infinity
  • -2 / 0 -> -Infinity
  • typeof Infinity -> "number"

Hexadecimal y Otras Bases

Si un número empieza con 0x, es hexadecimal:

  • 0xFF -> 255
  • toString(base) permite mostrar en otras bases:
  • 32.toString(2) -> "100000" (binario)
  • 32.toString(8) -> "40" (octal)
  • 32.toString(16) -> "20" (hexadecimal)

Números como Objetos

Se pueden crear objetos Number (pero no es recomendable):

  • let x = 123; // normal
  • let y = new Number(123); // objeto

Comparaciones:

  • x == y -> true
  • x === y -> false
  • Comparar dos objetos siempre da false (===).
